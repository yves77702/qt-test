1. runtime alignment
   继续缩小 `v91` execution runtime 与 `v540/v590` clean baseline 的差距。

2. dynamic exit runtime evidence
   继续收集 `dynamic_exit_runtime_shadow_sidecar.csv`，观察 `CASH_RELEASE_PROTECT_REVIEW` 与 `BLOCK_ACTIVE_WINNER_REVIEW` 是否稳定出现。

3. dynamic exit field completeness
   下一刀优先补 entry/cost 与 post-trigger sector context。字段缺失时只能 block，不能填假默认值。

4. dynamic exit outcome review
   outcome 只能进入 review-only 区，不能进入 runtime gate。重点看现金释放、卖飞、次日风险、候选替代队列。

5. holding-day T evidence only
   不把 completed buyback 当成功。继续拆 partial sell、buyback、keep-reduced。

## 晋级条件

一个研究分支想从 evidence 进入 runtime shadow，至少需要：

- 只使用当前可见字段
- 2023/2024 train 有支持
- 2025 exam 不坏
- 2026 frozen 只诊断
- 样本厚度足够
- active winner protection 通过
- route/paper/live/training flag 明确为 false
- 能写入稳定 sidecar 或 logger

一个分支想从 runtime shadow 进入 paper contract，还需要：

- 连续多日 forward evidence
- no-route shadow 与实际 runtime 字段一致
- 成交、现金、T+1、100 股、可卖数量约束全部明确
- 卖飞率、错过率、回补成功率、现金复用收益被分开审计
